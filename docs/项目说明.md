# 项目说明

**rpa-task** 是多平台电商（Amazon、Temu、Mano、Otto 等）的 RPA 报表自动化项目：从 ERP / RPA 共享盘读取 Excel，写入 MySQL，再在库内分步计算订单 SKU 利润。

---

## 架构概览

```
共享盘 Excel
    │
    ▼
scripts/dataImport/run_batch.py     ← 导入流水线（import_batch 串联）
    │
    ▼
MySQL（sales_order_*、amz_transaction、temu_order_item …）
    │
    ▼
scripts/archive/run_batch.py        ← 利润计算流水线
    │
    ▼
sales_order_sku_profit（按 calc_node 追踪计算阶段）
```

可选：**Docker** 提供 MySQL、Redis、Python 运行环境；Python 容器内 **cron** 每天 8:00 发送心跳邮件。

## 两条核心流水线

### 1. 数据导入（`scripts/dataImport/`）

```powershell
python scripts\dataImport\run_batch.py
```

| 顺序 | 脚本 | 目标表 |
|------|------|--------|
| 1 | `order_shipped.py` | `sales_order_shipped` |
| 2 | `order_refund.py` | `sales_order_refund` |
| 3 | `order_returned.py` | `sales_order_returned` |
| 4 | `order_temu.py` | 回填 Temu 价格到发货表 |
| 5 | `amz_transaction.py` | `amz_transaction` |

- 批次号 `import_batch`（如 `20260616_120000`）由 `run_batch.lock` 管理，当日可复用。
- 去重：`line_hash` + UPSERT。
- 路径：`config/path_config.py`（共享盘 `\\Betohow\...`）。

详见 [scripts/dataImport/README.md](../scripts/dataImport/README.md)。

### 2. 利润计算（`scripts/archive/`）

**前置**：先完成 dataImport。

```powershell
python scripts\archive\run_batch.py
```

| 顺序 | 脚本 | 作用 |
|------|------|------|
| 1 | `profit_001_order_sku.py` | 发货表 → 利润表初始化 |
| 2 | `profit_002_order_market.py` | 回填市场区域/编码 |
| 3 | `profit_002_order_price.py` | Temu RPA 明细价格覆盖 |
| 4 | `profit_003_order_first.py` | 头程/关税重算 |
| 5 | `profit_004_order_delivery_amz.py` | Amazon FBA 派送费 |

`calc_node` 字段记录每行已完成的计算阶段（`init` → `first_leg` → …）。

**待实现**：`profit_004_order_delivery_mano.py`（Mano 派送费）、`profit_005_order_vat.py`（VAT）。

详见 [scripts/archive/README.md](../scripts/archive/README.md)。

## 与旧版（python/v1）的关系

旧方案在 `python/v1` 下用 Excel 文件串联 A～M 模块逐步计算报表，已退出主线。

当前方案：**数据入库 → 分步 SQL 更新**，支持批次重跑、行级去重，便于自动化与对账。
