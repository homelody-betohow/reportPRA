# 报表系统重构 - 部署指南

> **项目根目录**：`d:\www\rpaReport`

## 快速部署

```powershell
cd d:\www\rpaReport
pip install -r requirements.txt
# 编辑 config/db_config.json 填入局域网 MySQL 连接信息
python database\db_connection.py
```

| 配置项 | 说明 |
|--------|------|
| 配置文件 | `config/db_config.json` |
| 数据库 | 局域网 MySQL（当前库名 `rpa-report`） |
| 表结构参考 | `database/tables/*.sql` |

**Navicat 备注乱码**：连接编码设为 **UTF-8 / utf8mb4**，详见 [docs/Navicat中文备注乱码.md](docs/Navicat中文备注乱码.md)。

## 数据库配置

### 1. 连接信息

数据库使用**局域网共享 MySQL**，由服务器统一维护，本机不需要安装 MySQL。

编辑 `config/db_config.json`：

```json
{
  "host": "172.18.188.18",
  "port": 3309,
  "user": "你的账号",
  "password": "你的密码",
  "database": "rpa-report",
  "charset": "utf8mb4"
}
```

账号密码向数据库管理员申请。修改后无需重启服务，直接运行测试脚本即可。

### 2. 表结构

线上库已有完整表结构，本仓库 `database/tables/` 目录下按表导出了 DDL（每张表一个 `.sql` 文件），供查阅和对照，例如：

- `product_sku.sql` — 产品 SKU
- `platform_shop_config.sql` — 平台店铺配置
- `sales_order_shipped.sql` — 已发货订单
- …共 20 张表

新建环境或需要对照字段时，直接打开对应文件查看即可，**无需在本机执行建表**。

### 3. Navicat 连接

| 项 | 值 |
|----|-----|
| 主机 | `db_config.json` 中的 host |
| 端口 | `db_config.json` 中的 port |
| 用户名 / 密码 | 同上 |
| 数据库 | `rpa-report` |
| 字符集 | **utf8mb4** |

库名含连字符，手写 SQL 时必须写反引号：`` USE `rpa-report`; ``

## 常见问题

### Q1：数据库连接失败

**错误提示**：`pymysql.err.OperationalError: (2003, "Can't connect to MySQL server...")`

**解决方法**：

1. 确认已连接公司内网（VPN 或办公网络）
2. 检查 `config/db_config.json` 中 host、port、账号密码是否正确
3. 用 Navicat 或命令行测试同一组连接信息：
   ```bash
   mysql -h 172.18.188.18 -P 3309 -u 你的账号 -p
   ```
4. 若仍失败，联系管理员确认账号权限与防火墙规则

### Q2：中文乱码

**解决方法**：

1. Navicat 连接编码设为 **UTF-8 / utf8mb4**
2. 详见 [docs/Navicat中文备注乱码.md](docs/Navicat中文备注乱码.md)

### Q3：映射查询不到数据

**解决方法**：

1. 检查对应表是否有数据：
   ```sql
   USE `rpa-report`;
   SELECT COUNT(*) FROM product_sku;
   ```
2. 检查查询条件（大小写、空格等）是否正确
3. 确认数据是否已从 Excel 迁移到数据库

### Q4：Excel 映射和数据库映射结果不一致

**解决方法**：

1. 对比两种方式的结果差异
2. 检查数据迁移是否完整、Excel 版本是否与线上一致
3. 检查映射规则是否一致（如 keep='first' vs keep='last'）

### Q5：性能优化建议

1. 批量查询代替逐条查询：
   ```python
   # 不推荐
   for sku in sku_list:
       result = service.map_sku_to_product_id([sku])

   # 推荐
   result = service.map_sku_to_product_id(sku_list)
   ```

2. 常用查询字段确保有索引（见 `database/tables/*.sql` 中的 KEY 定义）
